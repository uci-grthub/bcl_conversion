Set up a cron job to monitor for CopyComplete.txt and trigger Snakemake (via pixi) in a
new tmux session named after the run's library.

The monitor_and_run_snakemake.sh script already handles everything: it reads data_dir and
library_name from snakemake_config_project.yaml, waits for CopyComplete.txt in data_dir,
and launches `pixi run all` inside a tmux session. pixi activation loads .env and sets
SNAKEMAKE_PROFILE=profiles/default, so no --profile flag or manual env sourcing is needed.

1. Ensure 'data_dir' and 'library_name' are set in snakemake_config_project.yaml, e.g.:
     data_dir: /staging/nextcloud/Miseqi100/20260626_SH00564_0020_ASC2231455-SC3
     library_name: iR011

2. Provision the environment once (creates .pixi/ from pixi.toml / pixi.lock):
     pixi install

3. Make the script executable:
     chmod +x /staging/nextcloud/testing_illumina/MiSeqi100/iR011/monitor_and_run_snakemake.sh

4. Add the following line to your crontab ('crontab -e') to check every hour:
     0 * * * * /staging/nextcloud/testing_illumina/MiSeqi100/iR011/monitor_and_run_snakemake.sh >> /staging/nextcloud/testing_illumina/MiSeqi100/iR011/monitor_and_run_snakemake.log 2>&1

Notes:
- cron runs with a minimal PATH. The script resolves pixi automatically, falling back to
  $HOME/.pixi/bin/pixi when pixi is not on PATH, so no PATH edits are required in crontab.
- DRAGEN must be available on the system PATH for the cron user (bcl-convert is not
  provisioned by pixi).
